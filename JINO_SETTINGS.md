## Для диагностики ошибки 500

Вместо SSH можно использовать:

### 1. Веб-консоль Jino.ru
В панели управления найдите раздел "Консоль" или "Web Console" и выполните:

```bash
# Проверить логи backend
journalctl -u ux-audit-backend -n 50 --no-pager

# Проверить статус сервиса
systemctl status ux-audit-backend

# Проверить переменные окружения
cat /opt/ux-audit/backend/.env
```

### 2. Проверить через HTTP API
Backend доступен через HTTP, можно проверить напрямую:

```bash
# Health check
curl http://53893873b619.vps.myjino.ru/health

# Тестовый запрос
curl -X POST http://53893873b619.vps.myjino.ru/api/audit \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}'
```
