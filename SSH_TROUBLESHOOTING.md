# Диагностика проблемы с SSH подключением

## Проблема
SSH подключение к серверу недоступно (таймаут на порту 49376).

## Возможные причины

### 1. Порт 49376 закрыт firewall
Jino.ru может блокировать SSH порты на уровне сети для безопасности.

**Решение:** Проверить настройки firewall в панели управления Jino.ru.

### 2. SSH сервис не запущен
SSH демон может быть не запущен на сервере.

**Решение:** Проверить статус SSH через панель управления или веб-консоль.

### 3. Неправильный порт
Порт 49376 может быть неправильным или изменен.

**Решение:** Проверить правильный порт в панели управления Jino.ru.

### 4. Firewall блокирует подключения
Локальный firewall на сервере может блокировать SSH подключения.

**Решение:** Проверить настройки UFW или iptables на сервере.

## Решения

### Вариант 1: Использовать веб-консоль Jino.ru
Если SSH недоступен, можно использовать веб-консоль в панели управления Jino.ru:
1. Войдите в панель управления Jino.ru
2. Найдите раздел "Консоль" или "Web Console"
3. Используйте веб-консоль для выполнения команд

### Вариант 2: Проверить настройки firewall в панели Jino.ru
1. Войдите в панель управления Jino.ru
2. Найдите раздел "Firewall" или "Брандмауэр"
3. Убедитесь, что порт 49376 открыт для SSH

### Вариант 3: Использовать стандартный порт 22
Попробуйте подключиться через стандартный SSH порт 22:
```bash
ssh root@53893873b619.vps.myjino.ru
```

### Вариант 4: Обратиться в поддержку Jino.ru
Если ничего не помогает, обратитесь в поддержку Jino.ru для проверки SSH доступа.
